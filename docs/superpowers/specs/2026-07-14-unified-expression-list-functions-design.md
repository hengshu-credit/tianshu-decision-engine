# 统一表达式、名单函数与全量内置方法设计

## 目标

将当前统一操作数能力升级为可递归的表达式平台，并把所有具有执行语义的字段选择位置接入同一套配置、依赖分析、类型推导、编译和运行时解析链路。同时扩展名单查询函数与名单变量、条件操作符和内置函数目录，修复字段选择器焦点、布局和属性面板宽度问题。新结构是本次实现后的唯一持久化与运行时格式，不增加旧字段、旧名单配置或旧 modelJson 的兼容分支。

本次交付必须满足：

- `LITERAL`、`PATH`、`REFERENCE`、`FUNCTION` 作为新表达式 AST 的正式叶节点继续使用；仓库内示例、测试夹具和验收数据统一改为新结构。
- 新引用继续以 `refId + refType` 作为唯一关联依据，不通过编码、名称或标签回溯关联。
- 用户输入的变量编码、模型编码、路径、导入字段名和大小写原样保留。
- 名单函数只在服务端执行；使用名单函数或名单操作符的规则不能以客户端本地模式运行。
- 不开放按任意 Java 类名反射实例化对象；仅提供安全的 Map、List 和同结构空实例能力。
- 不新增第三方依赖，复用现有 Vue 2、Element UI、QLExpress、FastJSON、函数目录和字段目录。

## 已确认的业务语义

### 名单字段与名单组合

名单查询接受一个或多个查询表达式，以及一个或多个名单库。每个“查询表达式 × 名单库”形成一个布尔命中单元，组合模式固定为四种：

| 编码 | UI 短标签 | 命中规则 |
|---|---|---|
| `ANY_FIELD_ANY_LIST` | 任一字段 / 任一名单 | 存在至少一个命中单元 |
| `ALL_FIELDS_ANY_LIST` | 全部字段 / 任一名单 | 每个字段至少命中一个名单，可以是不同名单 |
| `ANY_FIELD_ALL_LISTS` | 任一字段 / 全部名单 | 至少有一个字段命中全部名单 |
| `ALL_FIELDS_ALL_LISTS` | 全部字段 / 全部名单 | 每个字段都命中每个名单 |

变量和函数统一使用这四种定义。单字段、单名单是四种模式的自然退化形式。

### 名单返回结果

- `BOOLEAN`：返回 `true/false`。
- `NUMBER`：返回 `1/0`。
- 只支持 `BOOLEAN` 和 `NUMBER` 两种返回模式，不读取或写入 `hitValue/missValue`。
- 名单匹配方式继续支持精确在名单内、精确不在名单内、被名单内容包含和未被名单内容包含。

### 表达式编辑方式

采用全屏三栏公式画布：

- 左侧为字段、阈值、函数、操作符、Key/Index 和类型转换素材。点击素材即插入当前选中参数槽。
- 中间为可选择的表达式树和实时公式预览，支持多层嵌套、撤销、重做、复制、替换和删除节点。
- 右侧为选中节点的参数配置。函数根据元数据自动生成参数槽，并支持添加、删除和重新配置参数。
- 页面内保留紧凑表达式输入控件。简单字段和阈值可快速选择；点击 `ƒx`、选择带参数函数或编辑已有复杂公式时进入全屏画布。
- 应用表达式前必须通过参数完整性、引用有效性、返回类型和上下文能力校验。

## 方案比较与选择

### 方案一：继续在每个页面扩展字段选择器

改动表面较小，但函数参数、类型推导、表达式预览、依赖分析和错误处理会继续分散在各设计器、模型和变量页面，无法保证行为一致。

### 方案二：把全部配置塞进现有字段选择器弹层

可以复用当前弹层，但多层嵌套会造成面板拥挤，并放大嵌套 popover 的焦点和 `aria-hidden` 问题。

### 方案三：统一表达式 AST + 全屏三栏画布

页面只声明上下文能力，表达式编辑、编译、解析和依赖收集全部复用。简单操作仍由紧凑触发器完成，复杂操作进入全屏画布。

本次采用方案三。

## 统一表达式数据结构

保留现有四种节点并新增可组合节点：

| `kind` | 用途 | 关键字段 |
|---|---|---|
| `LITERAL` | 手输阈值 | `value`、`valueType` |
| `PATH` | 外部输入路径或已解析路径 | `value`、`refId`、`refType`、`resolved` |
| `REFERENCE` | 受管理字段、常量、模型输出 | `refId`、`refType`、`code`、`valueType` |
| `FUNCTION` | 函数调用 | `functionId`、`functionCode`、`args[]`、`valueType` |
| `OPERATION` | 一元、二元或可变参数运算 | `operator`、`operands[]`、`valueType` |
| `ACCESS` | 取 Map Key、对象属性或数组 Index | `target`、`accessType`、`accessor`、`valueType` |
| `CAST` | 显式类型转换 | `targetType`、`operand`、`valueType` |
| `ARRAY` | 由多个表达式组成的数组 | `items[]`、`valueType=LIST` |
| `LIST_QUERY` | 条件操作符中的服务端名单配置 | `listIds[]`、`itemTypes[]`、`matchMode` |

所有子节点继续使用同一表达式结构。`code`、`label` 和公式文本只作为展示快照；真实受管理引用仍只认 ID 和引用类型。

### 额度公式示例

以下公式可由 `FUNCTION`、`OPERATION` 和 `REFERENCE` 节点递归表达：

```text
amount = CEIL(max(4200,
  min((min(max(月成功还款额, 已使用额度), 9000) * 风险系数 * 0.3
       + 风险额度 * 0.5), 7000)) / 500) * 500
```

根节点是乘法；左操作数是 `CEIL` 函数；其参数是除法；除法左侧继续嵌套 `max/min` 和加乘运算。每个字段引用保存其稳定 ID。

## 前端组件架构

### 共享组件

- `OperandPicker.vue`：保留紧凑触发器和简单字段/阈值快速选择，新增公式入口和表达式回显。
- `ExpressionEditorDialog.vue`：全屏容器，管理草稿、校验、撤销/重做和应用/取消。
- `ExpressionPalette.vue`：按上下文和当前参数类型展示字段、函数、操作符、访问器和转换。
- `ExpressionCanvas.vue`：展示和选择递归表达式树，维护插入位置。
- `ExpressionNodeInspector.vue`：配置函数参数、操作符参数、Key/Index 和类型转换。
- `expression.js`：创建、克隆、规范化、显示、类型推导、遍历和前端预览编译。
- `expressionContexts.js`：集中定义不同业务位置允许的节点类型和期望返回类型。

不为各设计器复制表达式编辑逻辑。

### 上下文能力

| 上下文 | 能力 |
|---|---|
| 条件左值、条件右值 | 完整读取表达式 |
| 动作值、结果值、模型输入来源、默认值、输入/输出转换 | 完整读取表达式 |
| API/DB/名单变量入参提取和结果转换 | 完整读取表达式 |
| 赋值目标、模型输出目标 | 仅 `PATH` 或可写 `REFERENCE` |
| 脚本编辑器插入 | 在画布配置后插入编译公式，同时维护递归引用 ID |

写入目标不允许函数、常量或算术式，因为这些内容不能成为 QLExpress 赋值左值。复杂逻辑配置在写入值或输出转换一侧。

### 全量接入位置

- 决策表：条件列左右值、动作值。
- 决策树：节点/连线条件左右值、节点动作值。
- 决策流：节点/连线条件、动作赋值、函数调用、循环输入、三元、集合判断和模板片段。
- 规则集：条件左右值、动作值。
- 简单交叉表、简单评分卡：条件/区间输入和结果值。
- 复杂交叉表、复杂评分卡：维度来源、条件值、结果值。
- QL 脚本：从字段选择器构造并插入复杂表达式，同时保存表达式内的引用 ID。
- 模型详情：输入来源、缺失默认值、输入转换、输出转换和输出目标。
- 分流实验：生产组与测试组条件树左右值。
- 变量管理：计算变量、API 参数映射与结果提取、DB 参数与结果路径、名单查询字段。

纯展示或身份关联字段不变成表达式，例如项目选择、规则选择、模型选择、名单库选择、变量名称和编码。

## 字段选择器交互修复

### 函数参数

选择无参数函数时直接形成 `FUNCTION` 节点。选择有参数函数时：

1. 安全关闭字段选择 popover。
2. 打开全屏表达式画布并选中函数根节点。
3. 根据 `paramsJson` 自动创建参数槽，显示参数名称、类型和说明。
4. 参数槽可以选择字段、阈值、函数或其他表达式。
5. 支持添加、删除、替换参数；固定签名缺少必填参数时不允许应用。
6. 应用后原输入框显示完整公式摘要，悬停显示完整公式。

### `aria-hidden` 根因修复

当前 `closePopover()` 在点击 popover 内元素时同步隐藏 popper，并多次调用 `doClose()` 和直接修改 `display:none`，焦点仍停留在即将被 Element UI 标记为隐藏的后代节点。

统一修复为：

1. 记录触发器引用。
2. 关闭前若活动焦点位于 popper 内，先把焦点移回触发器；触发器不可聚焦时才调用 `blur()`。
3. 下一帧只关闭一次 popover，不再全局查找并隐藏所有 `.var-picker-popover`。
4. 函数选择器关闭完成后再把焦点移入全屏对话框。
5. 隐藏状态同步设置 `inert`，显示时移除；`inert` 作为交互保护，焦点迁移仍是主要修复。

浏览器验收要求控制台不再出现 `Blocked aria-hidden...`，键盘焦点也不能丢失。

### 左侧分类宽度

- 左侧分类栏宽度根据最长分类标签和未过滤总数的位数计算，并限制在统一尺寸范围内。
- 分类标签和计数均 `white-space: nowrap`，计数不参与压缩。
- 左侧始终展示未过滤总数；搜索命中数放在右侧结果区，因此搜索不会改变左栏宽度或高度。
- 面板打开期间分类栏宽度保持不变，普通变量达到 200 个时仍保持单行。

### 打开定位

- 空值打开时默认进入“手动输入 → 手输阈值”，并自动聚焦阈值输入框。
- `LITERAL`、`PATH`、`FUNCTION` 和其他复杂节点打开时恢复到对应类别或直接进入画布并选中根节点。
- `REFERENCE` 和已解析 `PATH` 优先按 `refId + refType` 定位；仅对未解析外部路径按原路径展示。
- 分组对象和模型输出自动展开父组、切换到目标分页并滚动到当前字段。

## 设计器属性面板宽度

决策树和决策流复用同一个宽度计算工具：

- 最小宽度保持 320px。
- 最大宽度为当前设计器可用内容区域宽度的 80%，不再使用固定 1080px。
- 拖拽和浏览器尺寸变化时均重新计算并夹紧当前宽度。
- 画布剩余区域保持可用，不允许属性面板越界覆盖工具栏。

## 条件操作符目录

操作符元数据集中维护标签、适用类型、右值要求、右值类型和编译策略。表达式画布推导左值返回类型后刷新操作符；若原操作符不再适用，切换到该类型默认操作符并清理不兼容配置。

### 所有类型通用

- 等于、不等于、为空、不为空、任意。
- 在候选数组内、不在候选数组内。
- 在 Map Key 内、不在 Map Key 内、在 Map Value 内、不在 Map Value 内。
- 在名单内、不在名单内。右侧使用 `LIST_QUERY` 配置一个或多个名单，标记为仅服务端执行。

### 字符串

- 包含、不包含、以指定内容开头/结尾及其否定形式。
- 正则匹配、正则不匹配。
- 为空字符串、非空字符串。
- 在列表中、不在列表中。

### 数字和日期

- `>`、`>=`、`<`、`<=`、介于、不介于。
- 在列表中、不在列表中。

### 布尔和枚举

- 布尔：等于、不等于、为 true、为 false、为空、不为空。
- 枚举：等于、不等于、在枚举集合中、不在枚举集合中。

### List、数组和集合

- 包含元素、不包含元素。
- 包含任一元素、包含全部元素、不包含任一元素。
- 任一元素包含指定内容、任一元素不包含指定内容。
- 包含元素以指定内容开头/结尾及其否定形式。
- 空集合、非空集合、集合长度等于/大于/小于。

### Map 和对象

- 包含 Key、不包含 Key、包含 Value、不包含 Value。
- 空映射、非空映射。
- 对象只提供为空、不为空和通过 `ACCESS` 取字段后的类型化比较。

前后端使用相同操作符编码；前端预览和后端编译必须由成对测试锁定。

## 名单函数与名单变量

### 服务端名单函数

新增 Spring Bean 内置函数，复用 `RuleListService.match/matchAt`：

- `listHit(values, listIds, combinationMode, itemTypes, matchMode)`：返回布尔值。
- `listHitNumber(values, listIds, combinationMode, itemTypes, matchMode)`：返回 `1/0`。

`values` 接受单值、数组或 `ARRAY` 表达式；`listIds` 接受单 ID 或 ID 列表。函数目录参数元数据为名单 ID 使用专用名单选择器，为字段值使用完整表达式编辑器。调用日志记录字段数量、名单 ID、组合模式、每个命中单元和最终结果，但敏感字段值按现有日志策略处理。

函数通过服务端 Bean 注册，不进入客户端本地内置注册表。规则字段分析发现 `listHit`、`listHitNumber`、`LIST_QUERY` 或名单变量依赖时：

- UI 显示“仅服务端执行”。
- 保存/发布校验拒绝客户端本地执行模式并给出明确原因。
- 规则通过服务端测试和执行入口运行。

### 名单变量配置

新增配置：

```json
{
  "listIds": [1, 2],
  "queryOperands": [
    { "kind": "REFERENCE", "refId": 10, "refType": "VARIABLE" },
    { "kind": "REFERENCE", "refId": 11, "refType": "VARIABLE" }
  ],
  "combinationMode": "ALL_FIELDS_ANY_LIST",
  "matchMode": "IN_LIST",
  "itemTypes": ["MOBILE", "ID_CARD"],
  "returnMode": "NUMBER"
}
```

该 JSON 是名单变量的唯一配置结构。`listId`、`queryField`、`queryPath`、`field`、`hitValue` 和 `missValue` 均视为已移除字段；缺少 `listIds`、`queryOperands`、`combinationMode` 或 `returnMode` 时直接给出明确校验错误，不做默认补齐。

新 UI 使用多选名单下拉、可增删查询字段行和一个组合模式下拉。组合模式显示上述四个短标签，右侧带帮助图标；下方只显示当前模式的一句动态说明，保持界面简洁。

每个查询字段行点击时打开统一字段/表达式选择器。保存时至少需要一个名单和一个有效查询表达式。

### 运行时和依赖

`VariableSourceResolver` 递归解析 `queryOperands`，构建字段 × 名单命中矩阵，再按组合模式归并。依赖收集递归读取每个表达式，因此规则测试 Schema、执行参数绑定和血缘均能看到真实输入字段。运行日志展示四种模式及命中矩阵摘要。

## URule 方法覆盖

保留现有函数，并按参考页面补充以下安全语义等价方法。函数管理页面继续使用英文编码、中文名称和参数说明。

### 当前规则

- `currentRule`：返回当前外层规则的只读描述 Map。
- `currentRuleName`：返回当前规则名称。
- `currentMatchedConditions`：返回当前执行追踪中已匹配条件的只读集合；无追踪数据时返回空集合。

执行入口通过线程级运行时上下文传递规则 ID、编码、名称和匹配条件，并在 `finally` 中清理。

### 日期

- 现有：`dateFormat`、`dateConvert`、`dateAdd`、`dateSub`、`dateDiff`、`dateToMillis`、`millisToDate`。
- 当前时间：`currentDate`、`currentDateTime`。
- 加减：`dateAddYears/Months/Days/Hours/Minutes/Seconds` 与对应 `dateSub...`。
- 取值：`dateYear`、`dateMonth`、`dateWeekday`、`dateDay`、`dateHour`、`dateMinute`、`dateSecond`。
- 差值：`dateDiffMillis/Seconds/Minutes/Hours/Days/Weeks`。
- 月份天数：`dateDaysInMonths`、`dateDaysOutsideMonths`。

### 字符串

- 现有：去空白、长度、包含、正则替换、正则提取、切分和拼接。
- 新增：`strSubstring`、`strSubstringFrom`、`strSubstringTo`、`strLower`、`strUpper`、`strCharAt`、`strIndexOf`、`strLastIndexOf`、`strReplaceLiteral`。

### 数学

- 保留精确加减乘除、舍入、绝对值、幂和区间判断。
- 新增：`numMax`、`numMin`、`numSin`、`numCos`、`numTan`、`numCot`、`numLn`、`numLog10`、`numCeil`、`numFloor`、`numRoundInteger`。

### List

- 保留长度、按下标取值、首尾元素、去重、排序和包含判断。
- 新增：`arrMax`、`arrMin`、`arrAdd`、`arrSortBy`、`arrPluck`、`arrRemove`、`arrIsEmpty`。
- 修改型方法返回修改后的集合，便于在表达式和动作赋值中继续使用。

### Map

- 新增：`mapPut`、`mapRemove`、`mapHasKey`、`mapGet`、`mapSize`、`mapKeys`、`mapValues`、`newMap`。
- 修改型方法返回修改后的 Map。

### 对象与循环

- 保留对象路径取值、存在判断、大小、Key/Value 集合和 JSON 转换。
- 新增 `newList` 和 `newLike`；`newLike` 仅对 Map、List、Set 和数组创建同结构空实例。
- 不实现传入任意类名的反射实例化，这是已确认的安全边界。
- “中断循环”作为 `foreach` 内可用的动作块，编译为 QLExpress `break`；不伪装成可在任意位置调用的普通函数。

所有目录函数均通过契约测试验证元数据参数数量、Java/Bean 方法签名和真实执行结果。

## 编译、运行时与出入参解析

### 核心编译

扩展 `OperandCompiler` 为递归表达式编译器：

- `OPERATION` 按优先级总是用必要括号编译，避免嵌套语义变化。
- `ACCESS(KEY)` 编译为安全对象取值函数，`ACCESS(INDEX)` 编译为数组取值函数。
- `CAST` 编译为明确的类型转换内置函数。
- `ARRAY` 递归编译每个元素。
- `LIST_QUERY` 编译为服务端名单函数调用。

九种规则编译器、动作编译器和条件编译器继续调用同一个入口。

### 服务端运行时

扩展共享运行时表达式解析器，供模型字段、变量来源和函数测试使用：

- 支持嵌套 Map、扁平完整路径键、数组 Index 和对象 Key。
- 按函数 ID 调用函数并递归解析参数。
- 按声明类型执行转换；转换失败返回包含业务位置的错误。
- API 参数映射、DB 参数、名单查询字段和模型输入来源均使用同一解析器。
- 输出转换完成后再写入目标字段。

### 依赖、测试 Schema、血缘和日志

递归表达式访问器是唯一依赖入口：

- `LITERAL` 不产生依赖。
- `PATH/REFERENCE` 产生字段依赖。
- `FUNCTION/OPERATION/ACCESS/CAST/ARRAY` 递归访问子节点。
- `LIST_QUERY` 产生名单库依赖并标记服务端执行。

规则测试 Schema、字段分析、模型依赖展开、变量依赖排序、血缘和追踪展示全部复用该访问器。日志与追踪显示公式摘要、节点返回类型和解析后的参数，不把公式文本误识别成字段名。

## 新结构切换边界

- 不实现运行时旧结构归一化、双写或回退逻辑；新表达式 AST 和新名单配置是唯一输入。
- 四种基础操作数直接作为表达式叶节点，组合能力由 `OPERATION`、`ACCESS`、`CAST`、`ARRAY` 和 `LIST_QUERY` 提供。
- 仓库中的示例 SQL、测试夹具、默认模型和浏览器验收数据全部改为新结构，确保当前交付内容在同一格式上编译和执行。
- 现有函数编码和行为保持不变；新增方法使用新编码，避免覆盖同名不同签名函数。
- 项目仍处于未发布阶段，已有外部数据库中的旧配置需要重新配置，不在服务端保留透明兼容代码。
- 常量编译语义、模型输出 ID 关联和字段选择器分组行为保持不变。

## 错误处理

- 表达式缺少必填参数、包含空节点、受管理引用缺失 ID、类型不兼容或函数已停用时禁止应用，并定位到具体节点。
- 名单查询没有名单、没有查询字段或组合模式非法时禁止保存。
- 名单函数在客户端模式下禁止发布/执行并提示切换服务端模式。
- 正则表达式非法、Index 不是整数、Key 为空或除数为零时返回明确错误或按函数既有约定返回空值。
- 关闭画布时有未应用修改需要二次确认。
- 画布取消不修改页面绑定值；应用时一次性替换表达式，避免半成品写入 modelJson。

### 仓库完整校验

后端：

1. `mvn clean install -DskipTests`
2. 启动 `rule-engine-server` 并验证健康接口和登录接口
3. `mvn test`

前端：

1. `npm run dev`，确认实际监听端口和编译无错误
2. 浏览器逐项验收后停止本次启动的进程
3. `npm test`

同时执行 `git diff --check`，只处理本次变更引入的问题。

### 浏览器逐项验收

1. 变量管理创建名单变量：选择多个名单、添加多个查询字段、逐一验证四种组合模式、数字/布尔返回、保存回显和在线测试。
2. 函数管理测试名单函数、当前日期、当前日期时间及各新增分类的代表方法。
3. 字段选择器准备 200 个普通变量，验证分类单行、宽度稳定、搜索不抖动、空值聚焦阈值和已有字段定位。
4. 条件树分别选择字符串、数字、布尔、List、Map 字段，验证对应操作符和执行结果；名单操作符验证仅服务端提示。
5. 决策流用全屏画布配置并执行额度公式，保存、重新打开后公式、引用 ID 和结果一致。
6. 决策树、决策表、规则集、简单/复杂交叉表、简单/复杂评分卡和脚本编辑器分别完成一次表达式配置、保存、重新打开和测试。
7. 模型详情配置输入来源、默认值、转换和输出目标，执行模型测试并检查出入参。
8. 分流实验配置左右两侧复杂表达式并执行路由测试。
9. API、DB 和名单变量分别使用表达式入参/结果提取并执行在线测试，检查调用日志。
10. 决策树和决策流把右侧属性面板拖到接近页面 80%，确认画布和工具栏仍可操作。
11. 全程检查浏览器控制台，无 `aria-hidden`、未捕获异常或 Vue 错误。
12. 通过规则列表/测试页回归当前可执行规则，确认九种模型现有规则仍能编译和执行。

## 需求追踪

| 用户需求 | 设计落点 |
|---|---|
| 名单函数返回数字或布尔 | 服务端 `listHit/listHitNumber` |
| 名单变量多字段、多名单和组合模式 | 四模式命中矩阵、统一表达式字段行 |
| 函数无法选中、参数配置和删减 | 焦点修复、函数节点 Inspector |
| 选择器三位数换行和搜索抖动 | 稳定自适应分类栏 |
| 属性面板最大 80% | 共享宽度计算工具 |
| 类型化更多操作符 | 集中操作符目录和统一编译 |
| URule 全量安全方法及当前时间 | 内置函数扩展与安全边界 |
| 决策流复杂额度表达式 | 全屏递归表达式画布和 AST |
| 所有字段选择位置支持复杂配置 | 全量接入矩阵和统一出入参解析 |
| 默认阈值与已有字段定位 | 打开状态恢复和按 ID 定位 |
