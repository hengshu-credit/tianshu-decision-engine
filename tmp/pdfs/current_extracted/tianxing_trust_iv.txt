接口说明
 提交方式
采集接口请求方式为 GET,
请求头需添加内容类型为 Content-Type＝"application/json";charset="UTF-8"。
 授权码
调用时需使用授权码（accessToken），请参照《授权码获取接口》文档进行获取。
注：Http Status 401 为授权码时间已经过期；
Http Status 403 为授权码和机构不匹配，请注意判断。
 响应时间
接口超时时间为 30 秒。
 异常码
由于异常码较多，本文未对全部异常码进行陈列和说明。
如有需求，请参考《异常码文档》。
 调用样例
具体调用样例请参照同目录下的 demo 文件。
 环境说明
正式环境：正式账号使用，调用已开通接口无限制，按条数收费。
测试环境：测试账号使用，调用已开通接口受测试条数限制。我司未对返回结果进行
留存，每次请求均会调用真实数据并产生费用。如需进行接口测试，请与我司对接人
员联系，申请免费测试条数。
账号说明：正式账号仅适用于正式环境，测试账号仅适用于测试环境。
 计费说明
计费： 接口调用成功（返回结果 "success": true）
不计费： 接口调用失败（返回结果 "success": false）\n===== PAGE 4 =====\n4
可信度认证Ⅳ
一、 接口地址
 正式 URL: https://www.tianxingshuke.com/api/rest/riskTip/credCertify4
 测试 URL: https://test.tianxingshuke.com/api/rest/riskTip/credCertify4
 HTTP METHOD: GET
二、 请求参数
(一) 参数说明
参数名称 参数类型 参数说明 是否必填
account String 机构账号 是
accessToken String 授权码 是
idNo String 身份证号码 是
isEncrypt String
idNo 格式
1 表明文；
2 表 MD5 加密；
3 表 SHA256 加密;
4 表 SM3 加密
是
mockType String 模拟标识，本接口可传入
EXIST、NO_DATA。
否
注： mockType（模拟标识）仅在测试环境生效，且传入 mockType 后返回数据均为模
拟数据。并且通过此种方式调用接口不扣除测试条数。建议对接接口时使用此方式对接。
三、 响应数据
(一) 数据样例
1. 正常响应数据样例\n===== PAGE 5 =====\n5

(二) 响应值说明
1. 一级节点说明
字段名称 字段类型 字段说明
success boolean 请求状态（是否成功）
data 请求成功后，返回的数据（详见“data 节点说明”）
code string 异常码（接口异常时返回）
error string 异常说明（接口异常时返回）
errorDesc string 异常说明（中文）（接口异常时返回）
requestOrder string 唯一订单号
2. data 节点说明\n===== PAGE 6 =====\n6
字段名称 字段类型 字段说明
idCard string 身份证号码
caseType string
案件类型(CRIMINAL_RECORD、DRUG_RELATED、
DRUG_CRIME、RUN_CRIME、PERSONS_IN_CASE 、
CRIME_KEY、NORMAL)
typeDesc string 案件类型(前科、涉毒、吸毒、在逃、涉案、重点、正常)
status string 查询结果(EXIST、NO_DATA)
